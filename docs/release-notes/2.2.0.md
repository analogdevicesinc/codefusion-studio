# 2.2.0 Release Notes

<!-- markdownlint-disable MD007 -->

## Source

CodeFusion Studio source can be found on [:octicons-link-external-24: GitHub](https://github.com/analogdevicesinc/codefusion-studio/releases/tag/V2.2.0/) under tag `V2.2.0`{:target="_blank"}

## About this release

CodeFusion Studio 2.2.0 builds on the foundation established in previous releases, focusing on two key areas: **expanded platform support** and **AI workflow enhancements**.

## What's new

The following are some of the new features and enhancements introduced in version 2.2.0:

### AI workflow enhancements

AI capabilities have been enhanced with improved command-line integration, streamlined workspace creation, redesigned compatibility and profiling reports, and debugging improvements:

- **[`cfsutil ai`](../user-guide/cfsutil/ai/index.md) — AI tools integrated into cfsutil:**
    Completing the CLI-first workflow, the `cfsai` command-line tool has been migrated into `cfsutil` as the `cfsutil ai` command group, enabling end-to-end AI model workflows — from compatibility checking to code generation — entirely from the command line:
    - [`cfsutil ai build`](../user-guide/cfsutil/ai/build.md) — compile a model into deployable C/C++ source code.
    - [`cfsutil ai compat`](../user-guide/cfsutil/ai/compat.md) — check model compatibility with a target SoC and core.
    - [`cfsutil ai profile`](../user-guide/cfsutil/ai/profile.md) — profile model resource usage before deployment.
    - [`cfsutil ai model`](../user-guide/cfsutil/ai/model.md) — new in this release: add, list, update, and remove AI models within a workspace without editing `.cfsconfig` files manually.
    - [`cfsutil ai workspace create`](../user-guide/cfsutil/ai/workspace.md) — new in this release: generate a workspace directly from an AI model file with automatic compatibility checking and initial build.
    - [`cfsutil ai backends`](../user-guide/cfsutil/ai/backends.md) — list available AI backends and extension fields.

    `cfsai` remains available as a compatibility wrapper and continues to work for existing scripts.

    !!! note "Deprecated flag: `--json`"
        The `--json` flag has been deprecated across all `cfsutil ai`, `cfsutil elf`, and `cfsutil socs` commands. Use `--format json` instead. The `--json` flag continues to work but displays a deprecation warning and will be removed in a future release. The new `--format` flag accepts `text` (default) or `json` values and provides consistent output formatting across all commands.

- **[Workspace from AI Model](../user-guide/workspaces/create-workspace-from-ai-model.md) — Streamlined AI workspace creation:**
    - Create AI-ready workspaces directly from a model file with automatic compatibility checking and hardware configuration.
    - New wizard guides you through model file selection, SoC/board selection, and per-core targeting.
    - Generated workspaces include pre-configured AI application with model integration and sample data handling.
    - [AI Hardware Profiling view](../user-guide/workspaces/deploy-and-profile-ai-model.md) provides guided deployment to hardware with build status tracking and one-click model deployment.

- **[Embedded AI Tools](../user-guide/tools/manage-ai-models.md) enhancements:**
    - UX enhancements to the Compatibility and Resource Profiling report outputs, making model resource and performance estimates and compatibility results easier to visualize and interpret.
    - Improved access to Compatibility and Resource Profiling reports, with a new **Open Report** option on the CFS Home page.

- **[AI Debug Assistant](../user-guide/debugging/debug-tools/ai-debug-assistant/index.md) — improvements:**
    - General bug fixes and stability improvements. We recommend using the MCP server for production workflows for maximum portability and future compatibility.

- **[Profiling](../user-guide/tools/profiling.md) — streamlined Zephelin trace capture workflow:**
    - A new **Trace Capture** panel in the CFS Home Page provides a streamlined interface for capturing profiling traces. Configure trace settings once, then start and stop capture with a single click.
    - Enhanced **Profiling Configuration** in System Planner now displays cores individually with expandable configuration forms, making it easier to configure profiling options for multi-core workspaces.
    - Trace capture automatically generates both `.ctf` (Common Trace Format) and `.tef` (Trace Event Format) files, eliminating the need for manual conversion.
    - The [:octicons-link-external-24: Zephelin Trace Viewer](https://marketplace.visualstudio.com/items?itemName=Antmicro.zephelin-trace){:target="_blank"} extension is automatically installed as a dependency of the CodeFusion Studio extension and lets you visualize trace data directly within VS Code, providing an integrated alternative to the web-based viewer.
    - New profiling options include **CPU Load** and **Memory Usage** sampling and **Application Callgraph** for capturing function call graphs and application-level tracing.

### Expanded platform support

The following platform and tooling improvements expand CodeFusion Studio's reach across additional hardware and workflows:

- **[`cfsutil`](../user-guide/cfsutil/index.md) — CLI-first workspace and build workflows:**
    It is now possible to set up a workspace, build, and flash a project entirely from the command line, without opening the IDE. Use these commands in sequence:
    - [`cfsutil socs list`](../user-guide/cfsutil/socs.md#list) — list available SoCs.
    - [`cfsutil socs info`](../user-guide/cfsutil/socs.md#info) — display available boards, packages, and cores for a specific SoC.
    - [`cfsutil workspace configure`](../user-guide/cfsutil/workspace.md#configure-workspace) — generate a `.cfsworkspace` configuration file from command-line options. Previously this required the IDE.
    - [`cfsutil workspace create`](../user-guide/cfsutil/workspace.md#create-workspace) — create the workspace from a configuration file or from command line arguments.
    - [`cfsutil tasks list`](../user-guide/cfsutil/tasks.md#list-tasks) and [`cfsutil tasks run`](../user-guide/cfsutil/tasks.md#run-a-task) — discover and run build and flash tasks.

- **[Memory Viewer](../user-guide/debugging/debug-tools/memory-viewer/index.md) — multicore-aware memory inspection:**
    - The Memory Viewer is a new debug tool that lets you inspect device memory during live and retrospective debug sessions. It integrates directly with the CodeFusion Studio debug infrastructure, providing a native alternative to the Cortex-Debug memory viewer.
    - Displays memory in a scrollable hex and ASCII grid with configurable byte grouping (1, 2, or 4 bytes), column count (4–64), endianness (big/little), and display format (hexadecimal or decimal).
    - Navigates to any address by entering a hex or decimal value in the toolbar.
    - Auto-refreshes when the debugger halts and marks data as stale when the target resumes. Supports manual refresh when already halted.
    - Tracks all active debug sessions simultaneously and updates automatically when you switch context in a multi-core environment.
    - Supports retrospective debugging against core dump sessions in read-only mode.

- **[Core Dump Analysis](../user-guide/debugging/debug-tools/core-dump-analysis/index.md) — expanded SoC support:**
    - Core dump support has been expanded to nine additional MAX devices: MAX32650, MAX32655, MAX32660, MAX32662, MAX32666, MAX32670, MAX32672, MAX78000, and MAX78002.
    - Core dump analysis now supports a total of 12 SoCs. For details on enabling and using core dump support, see the [Core Dump Analysis documentation](../user-guide/debugging/debug-tools/core-dump-analysis/index.md).

- **[System Planner](../user-guide/tools/index.md) improvements:**
    - Enhanced **Pin Config** search for faster and more precise identification of pins, signals, and peripherals, with categorized results and navigation in the Pin Mux view.
    - **[Automated version resolution](../user-guide/installation/package-manager/install-required.md)** — when opening a `.cfsconfig` file with missing data models or plugins, System Planner displays a resolution screen with automated installation options.
    - Added full System Planner support (Memory Allocation, Peripheral Allocation, Pin Config, and Registers) for MAX32660, MAX32662, MAX32666, MAX32672, and MAX32675C microcontrollers.

- **[Workspace templates](../user-guide/workspaces/create-new-workspace.md) — expanded template library:**
    This release adds new templates and extends existing templates to additional boards.

    - **New templates include:**

        - **MSDK**: Library generation, Library usage, Watchdog
        - **Zephyr**: Basic threading, Button handling, POSIX philosophers, UART echo

        These templates are available across a range of supported processors, including MAX32655, MAX32657, MAX32658, MAX32666, MAX32672, MAX32690, and MAX78000.

    - **Board support expanded for existing templates:**

        Existing MSDK templates (DMA, Hello World (C++), I2C scan, RTC, UART Loopback) now support MAX78000. Existing Zephyr templates (ADC sequence, I2C custom target, LittleFS, Watchdog) now support MAX32666 and MAX78000.

        Templates are shown in the [Workspace Creation wizard](../user-guide/workspaces/create-new-workspace.md) based on your selected device — choose your processor and board to see available templates for your hardware.

- **Workspace Creation Wizard — SoC selection improvements:** Enhanced SoC selection page with expandable family cards, sticky search with match count and highlighting, and improved responsive layout.

- **Expanded SoC and RTOS support:**
    - Added support for Zephyr 4.3.0 across all supported MAX32xxx and MAX78xxx microcontrollers.

- **[Package Manager](../user-guide/cfsutil/package-manager.md) enhancements:**
    - Some packages require license acceptance before installation. When installing packages through either the VS Code Command Palette or `cfsutil`, a license acceptance prompt appears during installation. In the Command Palette, review the license agreement and any referenced URLs, then select **Accept** (or **Accept All**, if multiple licenses are listed) to proceed. In the command line, `cfsutil` prompts for confirmation using an interactive `y/n` prompt. For automated workflows, the new `--acceptLicense` flag on [`cfsutil pkg install`](../user-guide/cfsutil/package-manager.md#install) accepts package license agreements automatically, allowing installation in scripts and CI pipelines without interactive prompts.
    - A new **(CFS) Delete Package from Cache** command in the Command Palette lets you delete packages that have been uninstalled but remain in the local cache. Multiple cached packages can be selected for deletion at once.

- **[Plugins SDK](../user-guide/plugins/develop-plugins.md) reorganization:**
    The Plugins SDK is now distributed as the `cfs-plugins-sdk` package in the CodeFusion Studio repository, providing better version alignment with CFS releases and clearer dependency tracking. For complete documentation on developing plugins, including API reference and templating guide, see the [:octicons-link-external-24: CFS Plugins SDK README](https://github.com/analogdevicesinc/codefusion-studio/blob/main/packages/cfs-plugins-sdk/README.md){:target="_blank"}.

- **[`cfsutil` SoC command updates](../user-guide/cfsutil/socs.md) — updated output format:**
    The previous `{name}-{package}` format used to identify SoCs failed for parts with hyphens in their names, such as `adsp-21836`. The output format has been updated to separate SoC name and package explicitly. `socs export` now takes the SoC name as a positional argument with a separate `-p, --package` flag; `socs list` now groups packages per SoC.

    !!! warning "Breaking change"
        The default output format of `socs list` (text and JSON) and the usage of `socs export` have changed. If you have scripts that rely on the previous format, use the `--legacy` flag for `socs list` or the deprecated `-n, --name` flag for `socs export` while you migrate. Both will be removed in a future release.

## Known issues and limitations

### Plugins

**MSDK Project Plugin incorrectly appears for MAX32657:** When you use the [**Manually configure the workspace**](../user-guide/workspaces/create-new-workspace.md#manually-configure-the-workspace) option in the Workspace Creation Wizard to create a workspace for the MAX32657, the **MSDK Project Plugin** appears in the **Code Generation Plugin** selection screen even though this processor does not support MSDK. Do not select the **MSDK Project Plugin** in the **Code Generation Plugin** step for this processor. For processors with full MSDK support, see [supported processors](../user-guide/about/supported-processors.md).

**TrustZone manual configuration removed for MAX32657/MAX32658:** The manual TrustZone configuration option in the Workspace Creation Wizard has been removed for MAX32657 and MAX32658 devices due to build issues in manually configured TrustZone workspaces. [TrustZone support](../user-guide/workspaces/targets/trustzone.md) is still available through the **TF-M Secure Partition template**, which has been verified to build correctly. Use the template-based workflow for all TrustZone projects on these devices.

### Package Manager

**Package version conflict with cfs_zephyr_max32657_plugins/1.1.0:** Installing `cfs_zephyr_max32657_plugins/1.1.0` when `cfs_base_plugins/2.0.0` is already installed may result in a version conflict error: `Version conflict: Conflict between cfs_base_plugins/[^2.0.1] and cfs_base_plugins/2.0.0 in the graph.` The Package Manager does not automatically upgrade `cfs_base_plugins/2.0.0` to satisfy the plugin's dependency requirements. **Workaround:** Install `cfs_base_plugins/2.1.0` (or `cfs_base_plugins/2.0.1`) first, then install `cfs_zephyr_max32657_plugins/1.1.0`. Installing `cfs_zephyr_max32657_plugins/1.1.0` without first installing a compatible base plugins version may install `cfs_base_plugins/2.2.0`, which does not support the Zephyr CGM Demo and Zephyr BLE Peripheral workspace templates for MAX32657/8.

### Workspaces and builds

**Build error when workspace is opened in OneDrive or a folder with spaces:** A build error occurs when a valid workspace is opened in OneDrive or any location where the folder name contains spaces (for example, `My Workspace` or `OneDrive/My Documents`). This affects the CMake build process, which fails to handle paths containing spaces correctly. **Workaround:** Move the workspace to a folder path without spaces, or ensure the workspace is stored in a local directory path that does not contain spaces.

### CFS Command-Line (cfsutil)

**`cfsutil project create` fails for template-based workspaces:** Workspaces created from templates — whether using the Workspace Creation Wizard template flow (such as MSDK Multicore Hello World or Zephyr Single Core Blinky) or template/flag-driven `cfsutil workspace create` flows — generate a workspace configuration with an empty `Projects` array, causing `cfsutil project create` to fail with `Projects cannot currently be generated from this type of workspace file.` Use the [**Manually configure the workspace**](../user-guide/workspaces/create-new-workspace.md#manually-configure-the-workspace) option in the Workspace Creation Wizard, or run [`cfsutil workspace configure`](../user-guide/cfsutil/workspace.md#configure-workspace) followed by `cfsutil workspace create -i`, which uses an input `.cfsworkspace` file with a correctly populated `Projects` array.

### Profiling

**Profiling may fail with memory overflow:** Enabling Zephelin profiling options (CPU Load, Memory Usage, Application Callgraph, AI Model Profiling) increases memory usage and may cause build failures with linker errors (`region 'RAM' overflowed by X bytes`). Disabling some profiling options may reduce memory usage, but the device may not have sufficient memory to support profiling in the current application.

### Debugging

**J-Link Software and Documentation Pack V8.94 required for MAX32657/8:** As of this release, custom MAX32657/8 J-Link support files have been removed from the installer. Official support for these processors is now provided directly by Segger in J-Link Software and Documentation Pack V8.94 and later. **Minimum required version: V8.94**. Download the latest version from the [:octicons-link-external-24: Segger website](https://www.segger.com/downloads/jlink/){:target="_blank"}.

**J-Link flash erase may not clear full flash on MAX32657:** The **Erase (JLink)** action in the Actions panel may not fully erase flash memory on MAX32657 devices. Although J-Link reports successful completion, residual code may remain in flash, potentially causing unexpected behavior. As a workaround, use the **Erase (OpenOCD)** task instead, or run the following OpenOCD command:

```sh
openocd -s scripts -f interface/jlink.cfg -f target/max32657.cfg -c "init; reset halt; max32xxx mass_erase 0; shutdown"
```

**erase_JLink task may require cfs.jlink.path configuration:** The `erase_JLink` task (run using [`cfsutil tasks run`](../user-guide/cfsutil/tasks.md)) may fail with `JLinkExe: No such file or directory` if the `cfs.jlink.path` setting is not configured. **Workaround:** Set the `cfs.jlink.path` configuration setting to point to your J-Link installation directory (for example, `/opt/SEGGER/JLink_<version>` on Linux, `/Applications/SEGGER/JLink_<version>` on macOS, or `C:\Program Files\SEGGER\JLink_<version>` on Windows). For detailed instructions, see [Install Segger J-Link debugger drivers](../user-guide/debugging/debug-drivers/install-jlink-drivers.md).

**Debugging Zephyr projects with J-Link may not halt at `main`:** When using older versions of the Cortex-Debug VS Code extension or J-Link Software and Documentation Pack, J-Link debugger sessions for Zephyr projects may not halt at the `main` breakpoint on launch. This occurs because the `main` function is reached before the debugger completes its initialization sequence. **Solution:** Update to the latest Cortex-Debug extension (v1.12.1) and J-Link Software and Documentation Pack V9.36 or later to resolve this issue.

**J-Link has limited board support:** Not all debug functionality of the boards is accessible using the Segger J-Link. For a list of supported devices, see the [:octicons-link-external-24: Segger website](https://www.segger.com/supported-devices/jlink/maxim/){:target="_blank"}.
