## About this release

CodeFusion Studio 2.2.1 is an incremental release that builds on the foundation established in CFS 2.2.0. This release includes stability improvements, bug fixes, and new packages for Zephyr 4.4.0 development on MAX32657 and MAX32658.

## Zephyr 4.4.0 support for MAX32657 and MAX32658

To develop with MAX32657 and MAX32658 on Zephyr 4.4.0, install the `zephyr/4.4.0-b.1` package. The Zephyr ARM Toolchain package (`zephyr_arm_toolchain/1.0.1`) is installed automatically as a dependency. Install using **(CFS) Install Package** from the VS Code Command Palette or from the CFS Terminal by running:

```bash
cfsutil pkg install zephyr/4.4.0-b.1
```

The release also includes the **Zephyr Single Core PM GPIO Wakeup** workspace template, which demonstrates Zephyr power management on MAX32657/MAX32658 — runtime idle, standby, and suspend to RAM states with GPIO wakeup. The template requires the `zephyr/4.4.0-b.1` package to build. For installation steps and to get started with the new template, see the [MAX32657/MAX32658 Zephyr 4.4.0 tutorial](../tutorials/max32657-zephyr/index.md).

!!! warning
    Do not install the Zephyr 4.4.0 packages for MAX32657 and MAX32658 with earlier versions of CFS. These packages include features such as updated toolchain configuration that are not supported in earlier CFS versions.

## Bug fixes and enhancements

- **Workspace Creation Wizard — special character handling in SoC filter:**
    Fixed an issue where entering special characters (`\`, `^`, `$`, `*`, `+`, `?`, `.`, `(`, `)`, `|`, `[`, `]`, `{`, `}`, `/`, `-`) in the SoC filter field during workspace creation caused the display to crash. The filter now properly escapes these characters, preventing the UI from becoming unresponsive.

    !!! note "Wildcard characters not supported"
        The `*` character is not interpreted as a wildcard in the SoC filter. To find SoCs, enter partial names or identifiers without wildcards (example: `MAX32` to find all MAX32xxx devices).

- **Core Dump Analysis — removed `cfs.zephyrCoreDumpScriptsPath` setting:**
    The `cfs.zephyrCoreDumpScriptsPath` VS Code setting has been removed. CodeFusion Studio now resolves the Zephyr core dump scripts path automatically from the Zephyr toolchain, so no manual configuration is required. This change is backward compatible: if you previously defined this setting in your `settings.json`, core dump analysis continues to work, and the existing entry is simply ignored.

- **System Planner — I3C pin configuration (MAX32657/MAX32658):**
    Fixed an issue where enabling all I3C pins generated an undefined devicetree node for the Pull-Up Resistor (PUR) pin in the overlay files.

- **System Planner — automated version resolution:**
    Improved the [automated version resolution](../user-guide/installation/package-manager/install-required.md) experience when opening a `.cfsconfig` file with missing data models or plugins. The resolution screen now shows clear progress while it searches for compatible packages.

## Known issues and limitations

### Security

**Zephyr 4.4.0 security vulnerabilities:** Some security vulnerabilities addressed in Zephyr 4.4.0 remain under embargo. For details, see the [:octicons-link-external-24: Zephyr 4.4.0 release notes](https://docs.zephyrproject.org/latest/releases/release-notes-4.4.html#security-vulnerability-related){:target="_blank"}.

### Profiling

**Zephelin requires Zephyr 4.3.0:** To use [Zephelin profiling](../user-guide/tools/profiling.md), build with Zephyr 4.3.0, which is included in the installation and active by default. Zephelin profiling is not available on Zephyr 4.4.0 in this release.

### Debugging

**macOS only: Python 3.12 required for GDB Toolbox and Core Dump Analysis on MAX32657/MAX32658:** On macOS, debugging MAX32657 and MAX32658 with the `zephyr_arm_toolchain/1.0.1` package requires Python 3.12. To enable Python-based scripts, install Python 3.12 using Homebrew:

```bash
brew install python@3.12
```

If you are using GDB Toolbox scripts, also complete the steps in the entry below.

**macOS only: GDB Toolbox requires Python-enabled GDB:** On macOS, the Python-enabled GDB binary must be configured in your workspace's `.vscode/launch.json` to use GDB Toolbox scripts. To enable Python-based scripts, complete the following steps:

1. Open your workspace's `<core>/.vscode/launch.json`, where `<core>` matches your target's core (`m33` or `m4`). In the debug configuration, append `-py` to the `gdbPath` value in the `osx` section.

    For M33 targets (for example, MAX32657):

    ```json
    "osx": {
        "gdbPath": "${cfs:tool.path.arm.zephyr.eabi.toolchain.debuggerPath}-py"
    }
    ```

    For M4 targets (for example, MAX32690):

    ```json
    "osx": {
        "gdbPath": "${config:cfs.sdk.path}/Tools/zephyr-sdk/arm-zephyr-eabi/bin/arm-zephyr-eabi-gdb-py"
    }
    ```

2. Launch (or relaunch) a debug session.
3. Run a supported Python-based script (such as **Thread Analyzer**) from the GDB Toolbox panel to verify Python scripting is working. For instructions, see [Using the GDB Toolbox](../user-guide/debugging/debug-tools/gdb-toolbox/gdb-toolbox.md).

**Linux only: Python 3.12 issues for GDB Toolbox and Core Dump Analysis on MAX32657/MAX32658:** On some Linux systems, GDB Toolbox scripts and Core Dump Analysis for MAX32657 and MAX32658 may fail with `ModuleNotFoundError: No module named '_struct'`. In testing, reinstalling the Python 3.12 package resolved the issue:

```bash
sudo apt install --reinstall python3.12
```

CodeFusion Studio is supported on the following host operating systems:

- Windows 11 (64-bit)  
- macOS 15 and macOS 26 (ARM64)
- Ubuntu 22.04 and 24.04 (64-bit)

*Earlier or newer OS versions may work but are not officially supported.*
